T3: L2-028 秀恩爱分得快

题意

  • 给出多张照片的合影人员列表
  • 每张照片上的任两个异性之间的亲密度为1照片中的人数\frac{1}{\text{照片中的人数}}照片中的人数1​
  • 最后给定一对情侣
  • 输出这两个人最亲密的异性，如果有并列就按编号升序输出，如果互为最亲密的那就直接输出这俩人
  • 存在 -0 表示女性，0 表示男性。

思路

  • 毒瘤字符串模拟+小数精度
  • 坑 1 (性别判定)：必须用 string 读入，判断 s[0] == '-' 区分性别，使用int会在0的处理上出问题。
  • 坑 2 (连乘爆 long long)：如果用分数加法有可能一直无法约分导致分母很快就爆long long，同时因为数字很小(500)，使用小数精度不会丢失很多，所以使用 double 累加 1.0 / K。
  • 坑 3 (无关异性)：在一种比较极端的情况下，情侣中的男方可能没有和任何一个异性同时出现，这个时候每一个异性都应该是答案，也就是对于统计他的map需要把所有女性最初都置零，保证这个键的存在

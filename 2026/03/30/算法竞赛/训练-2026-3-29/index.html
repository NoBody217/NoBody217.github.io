T2: L2-029 特立独行的幸福

题意

  • 寻找区间 [L,R][L, R][L,R] 里的“特立独行的幸福数”。
  • 幸福数：一个数的各位数位平方和迭代最终能到达 1。如果陷入死循环（如 4），则不是
  • 特立独行：它不能是区间 [L,R][L, R][L,R] 中任何其他数在迭代过程中产生的中间数（即不依附于别人）。
  • 素数翻倍：输出时附带步数。如果是素数，步数 ×2\times 2×2。

思路

  • 全局污染标记：
    • 核心痛点在于判断“是否依附”。不要对每个数去查族谱，而是让每个迭代的数去“污染”它的后代。
    • 开辟全局数组 dep[N]。从 LLL 到 RRR 遍历所有数字 iii，无论其本身是否为幸福数，只要它产生了下一个数字 xxx，就标记 dep[x] = 1。
    • 迭代内部用 set<int> 记录出现过的数字，若某次迭代结果在 set 中已存在，立刻 return 防止死循环。
  • 然后对所有没被标记的判一下是不是幸福数，判的时候记录步数

T3: PAT甲级-1131 Subway Map

题意

  • 给定一张地铁线路图。对于多个查询，找出两站之间经过站点最少的路线。
  • 如果站点数并列，找出换乘次数最少的路线。输出总站数和具体的换乘方案（按站点打印 Take Line#X...）。

思路

  • fake做法：优先队列bfs
    • 反例:
      • 1号线：S A X
      • 2号线：S B X T
      • 两种到 X 的方式：
        • S -> A -> X，当前在线路 1
        • S -> B -> X，当前在线路 2
      • 这两条到 X 都是：距离 2，换乘 0，但是接下来 X -> T 只能走 2号线
      • 所以，如果你是从 2号线 到的 X，去 T 不用换乘，如果你是从 1号线 到的 X，去 T 要换乘一次
  • 正解：BFS建图 + DFS回溯
    • Step 1 (BFS 找所有最短路)：用 queue 跑无权图最短路。若 dis[v] > dis[u] + 1，则清空 pre[v] 并放入 u；若 dis[v] == dis[u] + 1，说明发现了一条等长的平行路线，将 u 追加进 pre[v]。运行完毕后，pre 数组构成了一张饱含所有最短路径的 DAG（有向无环图）。
    • Step 2 (DFS 暴力打擂台)：从终点 TTT 沿着 pre 数组递归回溯至起点 SSS。当到达起点时，拿到一条完整的最短路径。
      • 神级统计换乘法：将路径中相邻两点所属的 Line ID 塞入 std::set。因为是最短路，经过的线路种类数即反映了换乘次数。
      • 更新全局最少换乘记录 msiz，并记录最优路径 ans。
    • Step 3 (打印)：将选出的 ans 路径翻转为正序。遍历路径，当相邻两站所在的线路 ID 发生变化时，打印前一段线路的起点和拐点。别忘最后一段到终点

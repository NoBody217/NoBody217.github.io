笔记-拓扑排序

发表于2026-03-17|更新于2026-03-17|算法竞赛
|总字数:8.1k|阅读时长:29 mins.分钟|浏览量:

概述

  • 拓扑排序用来解决任务调度、依赖关系解析等带有明确先后约束的问题.
  • 适用于有向无环图 (DAG, Directed Acyclic Graph).
  • 核心的思想是“剥洋葱”:每次只处理当前没有任何前置依赖的节点. 配合队列 (Kahn算法) 实现，整体时间复杂度为 O(N+M)O(N + M)O(N+M)（NNN 为点数，MMM 为边数）.

算法流程

  • 初始化:
    • 维护一个 in_deg (入度) 数组，建图时统计每个节点的入度（即有多少条边指向它）.
    • 扫描所有节点，将所有初始入度为 0 的节点塞入队列 q.
  • 取出队头 (Kahn 算法核心):
    • 从队列中弹出一个节点 u，将其加入答案序列 topo_order.
  • 消除依赖 (松弛):
    • 遍历 u 的所有出边指向的邻居 v.
    • 因为 u 已经执行完毕，v 的前置依赖减少 1 个，即 in_deg[v]--.
    • 如果 in_deg[v] 减到了 0，说明 v 的所有前置任务已完成，将其塞入队列 q.
  • 重复以上步骤直到队列为空.

一些变式

  • 拓扑排序不仅用来生成序列，本身的过程天然具备图的性质验证和 DP 状态转移的功能.

验证给定序列是否合法

  • 不需要跑完整的 BFS，只需拷贝一份初始的 in_deg 数组.
  • 按给定序列依次遍历节点 xxx:
    • 如果 in_deg[x] != 0，说明前置条件未满足就被执行，直接判定非法.
    • 如果 in_deg[x] == 0，合法，并模拟执行 in_deg[v]--.

字典序最小/最大的拓扑序

  • 题目要求在多种合法的执行顺序中，优先执行编号最小的任务.
  • 做法:只需要将普通的 queue<int> 替换为优先队列 priority_queue (小根堆/大根堆).
  • 每次弹出的天然就是当前入度为 0 且编号极值的节点.

有向图判环 (死锁检测)

  • 如果图中有环 (如 A →\rightarrow→ B →\rightarrow→ C →\rightarrow→ A)，这些节点的入度永远无法减为 0，也就永远进不了队列.
  • 做法:算法跑完后，检查 topo_order.size() == n 是否成立.
  • 若 < n，则说明图中必然存在环/死锁，无解.

DAG 上的 DP / 关键路径

  • 除了排序，如果每个任务有耗时，求完成所有任务的最短时间.
  • 拓扑排序的过程就是天然的 DP 状态转移顺序.
  • 在执行 in_deg[v]-- 的同时，顺手进行状态转移:dp[v] = max(dp[v], dp[u] + time[u]).

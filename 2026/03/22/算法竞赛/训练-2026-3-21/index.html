T1: PAT甲级 1060 Are They Equal

题意

  • 给定两个数字字符串(可能包含前导零、纯小数、无小数点等各种形态),判断它们在保留 NNN 位有效数字的科学计数法下是否严格相等.

思路

  • 写模拟前一定先想好再开写
  • 最好不要写一堆分支判断,很容易变成屎山,工程化方法是更好的
  • 流水线做法:
    1. 统一格式:如果没有小数点 .,强行在字符串末尾加上一个 ..
    2. 找首位:寻找全串中第一个既不是 0 也不是 . 的字符下标 pos
    3. 如果找不到(pos == -1),说明是0,这里0也不要做特判,因为0可能给出多种形式(00.0,0.00,0…)
    4. 算指数:如果 pos 在小数点前面,指数为 dot - pos；如果在小数点后面,指数为 dot - pos + 1.纯 0 的指数强行记为 0.
    5. 截取有效位:从 pos 开始向后抓取最多 NNN 个数字(跳过小数点).如果抓不够,用 0 补齐后缀.最后拼接输出.

T3: L2-034 口罩发放

题意

  • 很长不想写了,自己去看吧
  • 极其恶心的大模拟.模拟连续 DDD 天的口罩发放业务,包含极其琐碎的现实规则.

思路

  • 依旧大模拟,尝试流水线工程做法
  • 要做如下几个事情:去除不合法的,找出最后生病的,每天的部分挑出能领到口罩的
  • 流水线架构:
    1. 筛出非法:写一个 valid 函数,严格校验身份证长度是否为 18 且全为数字 isdigit,合法的才存下来
    2. 找出生病:维护一个map映射id->状态,如果是第一次以生病的状态进来,就扔到生病区记录,这样保证每个人最多只记录一次
    3. 排序:按提交时间升序,时间相同按序号升序.
    4. 选人发放:用 map 记录上一次申请的时间,能发的就记录下来,最后输出.

T4: L3-017 森森快递

题意

  • 城市排成一条直线,相邻城市之间有道路容量限制.有一批快递订单,每单要求从城市 AAA 送到 BBB.问在不超过道路容量的前提下,最多能送多少件快递？

思路

  • 区间调度贪心 + 线段树维护区间最值.
  • 这种区间调度选最多的贪心方式都是先考虑右端点最靠左的,然后左端点降序,因为这样子是给后面的留下了更多的可能性
  • 然后题意区间覆盖选取受限于区间最小,选取后还要区间修改,所以考虑线段树
  • 线段树应用:
    • 线段树维护区间最小+change
    • 遍历排序后的订单,用线段树 query 查询该区间的“剩余容量最小值” VVV.
    • 将该订单送货量定为 VVV,答案加上 VVV.
    • 立即用线段树的 change 执行区间减法,将该区间的剩余容量全部减去 VVV(需使用 Lazy Tag).

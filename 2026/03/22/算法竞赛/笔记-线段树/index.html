笔记-线段树 (Segment Tree)

发表于2026-03-22|更新于2026-03-22|算法竞赛
|总字数:14k|阅读时长:50 mins.分钟|浏览量:

概述

  • 线段树用于解决区间查询(如区间求和、区间最值)与区间修改的数据结构.
  • 核心思想是基于分治 (Divide and Conquer),将一个大区间不断二分为两个子区间,形成一棵二叉树.
  • 有区间修改就需要有lazy tag,否则每次修改的复杂度可能退化到O(N∗log⁡N)O(N*\log N)O(N∗logN)
  • 配合Lazy Tag,可以使区间查询和区间修改的时间复杂度严格稳定在 O(log⁡N)O(\log N)O(logN).空间上通常需要开辟 4×N4 \times N4×N 的数组大小.

算法流程 (以维护区间最小值为例)

1. 向上更新 (pushup)

  • 作用：利用左右儿子的信息,更新父节点的信息.
  • 在 build 和 change 函数向回溯时调用.

2. 延迟下传 (pushdown)

  • 作用：当查询或修改需要进入子树时,将当前节点身上挂着的 Lazy Tag 传递给它的左右儿子.
  • 铁律：打标记时,必须同步更新该节点的真实值 t；下传时,不仅要传递 laz,也必须同步更新儿子的真实值 t.

3. 区间修改 (change)

  • 寻找目标区间 [x, y].
  • 致命避坑：决定是否向左右子树递归的条件是 目标区间 x 和 y 与 mid 的关系,绝对不能写成 l <= mid 或 r > mid,否则会导致无限递归爆栈.

4. 区间查询 (query / check)

  • 与 change 极其类似.
  • 致命避坑：求最小值的底层兜底返回值必须是极大值 (INF)；求最大值兜底为 −∞-\infty−∞；求和兜底为 0.

一些变式

  • 几何路段陷阱：在图论,常常给的是 NNN 个城市的坐标,这意味着只有 N−1N-1N−1 条路段.若处理从城市 LLL 到城市 RRR 的路网占用,实际操作的线段树区间应为 [L, R-1](或是映射后的边编号)
  • 求和变式：如果是维护区间和,pushup 变为 t[p] = t[p*2] + t[p*2+1].而在 pushdown 和 change 更新 t 值时,必须乘以区间的长度：t[p] += val * (r - l + 1).

笔记-树状数组

发表于2026-03-22|更新于2026-03-22|算法竞赛
|总字数:19k|阅读时长:1:08分钟|浏览量:

概述

  • 树状数组用于解决单点修改与前缀区间查询问题的高效数据结构.
  • 适用于所有满足可减性/可逆性的区间信息维护（最典型的是区间求和）.不能直接用于求区间最值 (RMQ).
  • 核心思想是利用数字的二进制特征（lowbit）对区间进行巧妙划分.查询和修改的时间复杂度均为极小常数的 O(log⁡N)O(\log N)O(logN)，空间复杂度仅为 O(N)O(N)O(N).

核心原理：lowbit 与管辖区间

  • lowbit(x) 的作用是取出数字 xxx 的二进制表示中，最低位的 1 以及它后面的 0 所构成的数值.
  • 位运算实现：x & -x.
  • 在树状数组 C 中，节点 C[x] 存储的是原数组 A 中从 x - lowbit(x) + 1 到 x 这段区间的总和,也就是长为lobit的区间

算法流程

1. 核心工具：lowbit

  • 用于计算管辖区间的长度

2. 单点修改 (add)

  • 作用：给原数组的第 xxx 个位置加上数值 vvv.
  • 逻辑：修改 xxx 会影响包含 xxx 的所有上级区间.上级区间的寻找方法是不断加上自身的 lowbit.
  • 边界：必须小于等于最大容量 NNN（注意 NNN 往往是值域的上限）.

3. 前缀查询 (query)

  • 作用：求原数组前 xxx 项的和（即 A[1]+A[2]+⋯+A[x]A[1] + A[2] + \dots + A[x]A[1]+A[2]+⋯+A[x]）.
  • 如果要查询任意区间 [L,R][L, R][L,R] 的和，利用前缀和的相减性质：query(R) - query(L - 1) 即可
  • 逻辑：将前 xxx 项的区间拆分成若干个已知的树状数组区间.子区间的寻找方法是不断减去自身的 lowbit.
  • 边界：必须大于 0.

一些变式与应用

变式 1：求动态序列的中位数 / 第 K 小

  • 痛点：序列动态增删元素，要求随时以 O(log⁡N)O(\log N)O(logN) 或 O(log⁡2N)O(\log^2 N)O(log2N) 的时间查出排在第 K 位的元素.
  • 正解：值域树状数组 + 二分答案.
  • 做法：
    1. 将树状数组 t 作为值域计数桶.当数字 xxx 入栈时，add(x, 1)；出栈时，add(x, -1).
    2. 此时 query(M) 的含义变成了：当前集合中，数值 ≤M\le M≤M 的元素总共有多少个？
    3. 要找第 KKK 小的数，只需在值域区间 [1,MAX_VAL][1, MAX\_VAL][1,MAX_VAL] 内进行二分查找 mid.
    4. 若 query(mid) < K，说明 mid 取小了，目标在右半区 l = mid + 1.
    5. 若 query(mid) >= K，说明 mid 可能是答案或偏大，向左收缩 r = mid - 1.
    6. 二分闭环 while(l <= r) 结束后，l 即为最终答案.

变式 2：求逆序对

  • 这是树状数组的另一大经典应用.
  • 将输入序列从前往后遍历.对于当前的数字 A[i]A[i]A[i]，利用值域树状数组统计,前缀和就是小于等于x的数的个数.
  • 即：当前逆序对增加量 = 已经遍历过的数字总数 - query(A[i])(比自己小于等于的的数).
  • 统计完后，将 A[i]A[i]A[i] 扔进树状数组：add(A[i], 1).

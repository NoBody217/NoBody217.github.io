T2: L2-036 网红点打卡攻略

题意

  • 给定一张无向图。给出若干条访问路径，要求判断每条路径是否是一条合法的“网红点打卡攻略”。
  • 合法条件（本质上是求哈密顿回路）：
    1. 必须从老家（起点 0）出发，最后回到老家（0）。
    2. 必须且只能经过图中每一个非 0 节点恰好一次。
    3. 相邻打卡点之间必须有路相连。
  • 输出合法路径的个数，以及其中总花费（路径长度和）最小的路径编号和具体花费。

思路

  • 路径合法性校验
  • 校验流程：
    1. 长度拦截：如果给出的打卡点数量 t≠nt \neq nt=n，说明要么没走完所有点，要么走了重复的点，直接 continue 不合法。
    2. 全覆盖拦截：用 vis 数组记录走过的点，若存在 1∼n1 \sim n1∼n 中未被访问的点，直接不合法。
    3. 连通性与花费计算：依次累加起点 0 到 P1P_1P1​、中间各点、以及 PnP_nPn​ 到终点 0 的距离。在此过程中，如果某条边不存在（距离为 INFINFINF），则不合法。
  • 校验通过后，按要求进行 val < ans 的打擂台更新即可。

T3: L3-011 直捣黄龙

题意

  • 敌方大本营（终点）隐藏在由多个城市连成的无向图网络中。
  • 要求找出一条从起点到终点的路径，满足以下四重优先级的权值：
    1. 最短距离（路程必须最短）。
    2. 选解放的城市数量最多的。
    3. 如果城市数量也并列，选杀敌总数最多的。
  • 输出这条路径经过的所有城市名字、最短路径条数、最短距离、最大杀敌数。

思路

  • Dijkstra 多权值。
  • 降维预处理：题目给出的是字符串城市名，极其恶心。读入时直接用 map<string, int> 将城市名映射为 0∼N−10 \sim N-10∼N−1 的整数编号。并在 vector 中存下每个整数编号对应的字符串，彻底剥离字符串操作。
  • Dijkstra 核心松弛逻辑：
    • 开辟四个状态数组：dis (距离), cnt (城市数), kil (杀敌数), num (路径条数)，以及一个前驱数组 pre。
    • 如果 dis[v] > dis[u] + w：绝对碾压。更新 dis, cnt, kil, pre，且路径数继承 num[v] = num[u]。
    • 如果 dis[v] == dis[u] + w：打平，进入次级评判。
      • 首先，路径数必须累加：num[v] += num[u]。
      • 接着比城市数：如果 cnt[v] < cnt[u] + 1，更新 cnt, kil, pre。
      • 如果城市数依然打平 cnt[v] == cnt[u] + 1：最后比杀敌数，如果 kil[v] < kil[u] + a[v]，更新 kil, pre。
  • 路径打印：根据 pre 数组从终点回溯到起点，reverse 后根据映射表还原输出即可。

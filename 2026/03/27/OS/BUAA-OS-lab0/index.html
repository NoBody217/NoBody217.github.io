Linux操作

  • diff [op] <file1> <file2>

    • 比较不同
    • 常用选项:
      • -b:不检查空白字符
      • -B:不检查空行
      • -q:仅显示有无差异,不显示详细信息

  • find [path] <op>

    • 在给定路径下递归查找文件,输出路径,如不指定path则在当前目录下查找
    • 常用选项:
      • -name <file>:查找对应文件

  • grep [op] PATTERN FILE

    • 输出匹配PATTERN的文件和相关的行
    • 常用选项:
      • -a:不忽略二进制数据
      • -i:忽略大小写
      • -r:递归查找
      • -n:显示行号
    • 模式匹配
      • ^PATTERN以…开头的行
      • PATTERN$以…结尾的行

  • tree [op] [dir_name]

    • 输出文件树
    • 常用选项:
      • -d:只列出目录
      • -a:列出全部文件(包括隐藏的)

  • man [op] command

    • 查看手册
    • man test可以查看[]

  • chmod [ugoa] [[+-=][rwxX]] [file]

    • ugoa表示:拥有着,群组,其它人,所有人
    • ±=表示:增加,取消,唯一设定短线
    • rwxX表示:读取,写入,可执行,X只有当该文件是个子目录或者该文件已经被设定过为可执行
    • 可以用三个0~8的数字设定

  • sed [选项] '命令' 输入文本

    • 选项（常用）:
      • -n 安静模式,只显示经过 sed 处理的内容。否则显示输入文本的所有内容。
      • -i 直接修改读取的档案内容,而不是输出到屏幕。否则,只输出不编辑。
      • -e 允许多点编辑。例如sed -e 's/a/b/' -e 's/c/d/'一次性执行多条命令。
    • 命令（常用）:
      • [行号]a\ [内容] 新增,在行号后新增一行相应内容。行号可以是“数字”,在这一行之后新增,也可以是“起始行,终止行”,在其中的每一行后新增。当不写行号时,在每一行之后新增。使用 $ 表示最后一行。后面的命令同理。
      • [行号]c\ [内容] 取代。用内容取代相应行的文本。
      • [行号]i\ [内容] 插入。在当前行的上方插入一行文本。
      • [行号]d 删除当前行的内容。
      • [行号]p 输出选择的内容。通常与选项-n 一起使用。
      • s/re/string/将 re（正则表达式）匹配的内容替换为 string。
    • 例子

  • awk [选项] '条件 {动作}' 输入文件

    • awk的核心在于解决了数据按列排布的编
    • 选项（常用）
      • -F "分隔符" 指定输入的列分隔符。
        • 默认连续的空格或Tab键
    • 内置变量
      • $0 代表当前处理的这一行的完整原始文本。
      • $1,$2... $n 代表当前行被切分后的第 1 列、第 2 列… 第 n 列。
      • NF (Number of Fields):当前行一共被切成了多少列。
        • 【神级用法】 $NF 代表最后一列（不管这行有几列）,$(NF-1) 代表倒数第二列。
      • NR (Number of Records):当前处理的是第几行（行号）。
    • 条件
      • 【无条件（空）】:每行都做
      • 【正则匹配】:/正则表达式/。只有包含匹配内容的行才执行动作。
        • 例:awk '/ERROR/ {print $2}' log.txt （只有遇到包含 ERROR 的行,才打印它的第二列）
      • 【关系运算】:利用 ==,>,<,!= 进行数值或字符串比较。
        • 例:awk 'NR==2 {print $0}' a.txt （只打印第 2 行）。
        • 例:awk '$3 > 100 {print $1}' a.txt （只挑选出第 3 列数字大于 100 的行,并打印它的第 1 列）。
      • 【特殊条件:BEGIN 和 END】:
        • BEGIN:在处理第一行文本之前执行（常用于打印表头、初始化变量）。
        • END:在处理完最后一行文本之后执行（常用于打印最终的统计结果）。
        • 例:awk 'BEGIN {print "开始处理"} {print $1} END {print "处理完毕"}'
    • 动作
      • { print 变量 }:最常用的输出命令。用逗号隔开变量,输出时会自动加上空格。
        • 例:{print $1,$3} （打印第一列和第三列,中间带空格）。
        • 例:{print "名字是:" $1} （将自定义字符串与变量拼接打印）。
      • { printf "格式",变量 }:和 C 语言的 printf 完全一样！用于格式化对齐输出。
        • 例:{printf "PID: %05d\n",$1} （以 5 位数字补零的格式打印第一列,注意要手动加 \n 换行）。
      • 【高级编程动作】:awk 内部支持像 C 语言一样的 if-else、for 循环和数学运算。
        • 例:awk '{ sum = sum + $1 } END { print "总和是:",sum }' （把每一行的第一列数字累加起来,最后输出总和）。

  • head [op] [file]

    • 显示文件首部内容
    • 选项
      • -n <n>:显示前n行
      • -c <n>:显示前n字节

  • tail [op] [file]

    • 显示文件尾部内容
    • 选项
      • -n <n>:显示后n行
      • -c <n>:显示后n字节
      • -f:当文件增长时输出后续添加的内容

  • wc [op] [file]

    • 统计文本文件行数,单次数,字节数
    • 选项
      • -l:只显示行数
      • -w:只显示单词数
      • -c:只显示字节数

  • sort [op] [file]

    • 将标准输入按行排序输出
      • 选项
        • -k [num]:以指定列为关键字排序
        • -r:逆序排序

GCC

  • 编译->链接
  • gcc [option] [parameter]
    • 常用选项:
      • -o:指定生成的输出文件
      • -S:将C代码转化为汇编代码
      • -Wall:显示一些警告
      • -c:只编译不链接,生成.o文件
      • -M:列出依赖
      • -I<path>:指定头文件目录,可以用相对路径

makefile

  • makefile核心结构:

  • 自定义变量

    • 定义的时候直接用,使用的时候用$()获取变量内容

  • 自动化变量

    • $@:目标文件
    • $^:所有依赖
    • $<:第一个依赖

  • 伪变量

    • 不在检查是否存在,直接做对应的操作
    • 常用于clean操作

  • 模式规则

    • %.o:%.c:所有的.o文件由对应的.c文件生成

上机后

  • 借鉴一些大佬的内容
  • 补充一些琐碎的点
    • man test：可以查看比较符
    • 对于空串的判定：[[ -n "$errors" ]]
      • 首尾的空格不能少
      • -z判空，-n 判断非空
      • -z/-n必须放在前面
      • 对于变量和字符串的处理一定要用""括起来
  • grep: grep [OPTIONS] PATTERN [FILE…]
    • grep的返回值为0/1/2，0为找到，1为没找到，2为程序错误（文件不存在）
    • grep逐行读取输入（文件或标准输入），将每一行与给定的正则表达式进行匹配。如果匹配成功，默认会将整行内容输出到终端。
    • 重要参数：
      • -i 忽略大小写。
      • -v 反向查找：显示不包含模式的所有行。
      • -w 全字匹配：必须匹配整个单词，而不是单词的一部分。
      • -x 全行匹配。
      • -n 在输出的前面加上行号。注意区分这玩意与sed中-n的区别，一般grep里-n没啥用
      • -c 只统计匹配到的行数，不显示具体内容。
      • -l 只列出文件名（包含匹配内容的文件）。
      • -q 静默模式。不输出任何内容，仅通过返回值判断是否找到。
  • makefile的多文件递归调用
    • 通过$(MAKE)在子目录调用make
    • 通过-C切换目录，注意不同行的make命令属于不同进程，所以如果在第一行切换目录，在第二行执行会失败，需要写成cd test && make all

BUAA-OS-lab1

发表于2026-03-27|更新于2026-03-30|BUAA-OS
|总字数:13k|阅读时长:48 mins.分钟|浏览量:
  • 搞笑来的，完全无从下手
  • 生啃指导书+看别人的博客勉强下手

上机前

  • 指导书大概告诉了我们这么几个事情：

QEMU

  • 模拟器,已经帮我们实现了内存和外围设备的启动
  • 剩下的启动流程:加载内核到内存,跳到内核入口

Makefile

  • 课程组提供的内核给了一个非常复杂的结构，以及一个超绝makefile

ELF文件

  • 通过makefile编译链接之后得到了"操作系统内核"
  • 连接器通过ELF文件的结构最终构建成内核，内核本质也是一个ELF文件
  • 满足如下结构的文件
  • 最基本单位是节,若干个节组成一个段
  • 节负责链接,段负责装载
  • hexed.it

ELF头

  • 记录程序基本信息
  • 结构如下

段头

  • 段记录运行的时候用的信息
  • 32字节,8个字段,每字段四个字节

节头

  • 节记录编译和链接使用的信息
  • 40字节

内存布局-内核加载到哪

  • MIPS一共4GB，内核放在kseg0
  • 划分方式如下
  • MOS内存布局

Linker Script-控制内核加载地址

  • .text,.bss,.data这种比较重要的段要放到内核最开始的地方，保证一上电就能用
  • 记录各个节应该如何映射到段和各个段加载到哪个位置
  • 通过写.ld文件来规定加载到哪里

printk

  • 纯考C语言
  • 变长参数
    • va_list：声明
    • va_start(va_list ap,lastarg)：初始化，输入上一个参数，决定以什么形式读取数据
    • va_arg(va_list ap,类型)：以某种类型读取一个数据，并移动指针
    • va_end(va_list ap)：结束使用
  • 一些琐碎的实现

系统readelf用法

  • readelf [op] file
    • 常用选项
      • -h：查看ELF头
      • -S：查看节头表
      • -l：查看段头表
      • -s：查看符号表
      • -a：全打印

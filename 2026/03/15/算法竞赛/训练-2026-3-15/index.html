T1: L1-086 斯德哥尔摩火车上的题

题意

  • 给出两个字符串,分别计算相邻两个字符的 ASCII 码差值.
  • 如果两个字符串经过计算后,连续的差值序列（只保留差值为偶数的项）完全一致,则输出这个序列；否则输出指定的字符串.

思路

  • 纯模拟,手速题

T2: L1-093 猜帽子游戏

题意

  • NNN 个人戴着不同颜色的帽子.多轮游戏,每轮给出大家对自己帽子颜色的猜测（输入 0 代表放弃猜测）.
  • 获胜条件：团队中没有一个人猜错,且至少有一个人猜对.

思路

  • 语法题
  • 将真实颜色存在数组中.对于每一次查询,遍历 NNN 个人的猜测.
  • 维护几个变量统计次数,最后判断就行

代码

T3: L2-038 病毒溯源

题意

  • 找出一棵树中从根节点开始的最长链.
  • 如果有多条最长链,输出字典序最小的那一条.

思路

  • 先找根,维护一个fa数组然后从随便一个点开始往上找就行,或者直接找没有fa的
  • 然后跑一个dfs,为了字典序最小把边提前做个升序排序
  • 这样 DFS 遍历时,天然就是按照字典序从小到大访问的
  • 维护一个全局的 ans (记录最长链) 和局部的 path (记录当前走过的路径).
  • 当到达叶子节点时,只要 path.size() > ans.size() 就无脑替换 ans

T4: L2-016 愿天下有情人都是失散多年的兄妹

题意

  • 给定一张家谱图,查询两人能否结婚.
  • 条件：必须性别不同,且在 五代以内 没有公共祖先.

思路

  • 可以直接整个LCA,但是不会,因为五代以内,不是很多,可以考虑爆搜
  • 先对情侣中的一个搜5代,标记所有人
  • 然后从另一个人也向上找5代,如果遇到被标记的就说明不行
  • 写两个dfs维护,不要在一个dfs里干两件事,写一个得辨别逻辑优先级,麻烦

T5: L3-007 天梯地图

题意

  • 给定一张图,每条边有距离（Distance）和时间（Time）两个权值.
  • 要求分别求出：
    1. 最快路线（时间最短）.如果时间相同,取距离最短的路线.
    2. 最短路线（距离最短）.如果距离相同,取途径节点数最少的路线.
  • 如果两条路线完全相同,合并输出；否则分别输出.

思路

  • 经典的 多权值Dijkstra.纯拼码力.
  • 开两套图 G1 和 G2,分别跑两遍 Dijkstra：
    • dij1 维护 dis1 (距离最短) 和 cnt (节点数最少).
    • dij2 维护 dis2 (时间最短) 和 len (距离最短).
  • 用 pre1 和 pre2 记录前驱,跑完后利用 while(cur != -1) 逆向回溯出路径并 reverse.
  • 最后判断两个 vector 是否相等来决定是合并输出还是分开输出,注意输出格式的小细节（=> 的拼接）.

L1-4 四项全能

题意

  • 全班 nnn 个人，有 mmm 项技能，每项技能分别有 kik_iki​ 个人会。求至少有多少人 mmm 项全都会。

思路

  • 容斥原理变体（逆向思维）。
  • 不算“会的人”，算“不会的人”。第 iii 项技能有 n−kin - k_in−ki​ 个人不会。
  • 假设这些“不会的人”完全不重合，那么“至少不会一项技能的人”最多有：S=∑(n−ki)S = \sum (n - k_i)S=∑(n−ki​) 个。
  • 此时，“什么都会的人”最少就是 n−Sn - Sn−S 个。
  • 注意处理负数

L1-5 别再来这么多猫娘了

题意

  • 给定多个违禁词，按输入顺序依次在文本中查找并替换为 <censored>。
  • 如果替换总次数超过阈值 KKK，则输出替换次数并附加警告语。

思路

  • 如果直接 replace 替换成 <censored>，而原句中后续又出现了包含在这 10 个字符里的违禁词，会导致无限套娃死循环或破坏已替换的占位。
  • 占位符流水线：
    1. 遍历违禁词数组。
    2. 使用 string::find 从左向右查找，每次从上一次找到的位置 pos 继续往后找，避免回头。
    3. 找到后，将违禁词替换为不可见字符 \1（长度为1），并记录替换次数。
    4. 所有违禁词处理完毕后，再将文本中所有的 \1 统一替换为 <censored>。

L2-049 鱼与熊掌

题意

  • 给定 NNN 个人对 MMM 种物品的拥有关系。
  • 随后给出多次查询，每次查询给定两种物品 AAA 和 BBB（如鱼与熊掌），问这群人中，有多少人同时拥有这两种物品。

思路

  • 集合求交集 / 哈希表查询。
  • 反向映射，对每个物品记录拥有他们的人
  • 可以直接用 vector<int> 不能用 unordered_set<int> 存储每个人拥有的物品。
  • 每次查询时，暴力遍历所有的人，检查其物品集合中是否同时包含 AAA 和 BBB。
  • 可以用 std::bitset 对每个物品记录拥有它的人（第 iii 个人拥有记为 1），查询时只需对 AAA 和 BBB 的 bitset 做一次按位与 &，然后 count() 即可瞬间出结果，速度更优

L2-050 懂蛇语

题意

  • 将一句话中每个字（拼音）的首字母提取出来，拼接成“蛇语”密钥。
  • 给定一个包含多条原句的字典，之后进行多次蛇语查询：
    • 如果查到，输出对应的原句。如果有多个原句对应同一个密钥，按字典序排序并用 | 隔开。
    • 如果查不到，原样输出该蛇语密钥。

思路

  • 字符串流处理与映射。
  • 使用 getline 读入带有空格的拼音长句。
  • 写一个gethead 获得一句话的所有首字母构成的串
  • 用 map<string, vector<string>> 存储映射关系
  • 读完字典后，遍历 map，对每个 vector 内部进行 sort 字典序排序。查询时直接查表即可。

L2-051 满树的遍历

题意

  • 给定一棵多叉树（以给出每个节点父节点编号的形式），根节点父编号用 0 表示。
  • 判断它是否是“KKK 阶满树”（要求所有非叶子节点的度数必须完全一致，且等于全树的最大度数 KKK）。
  • 输出该树的度数、是否为满树的判断（yes/no），以及前序遍历序列（要求兄弟节点按编号升序访问）。

思路

  • 树的遍历与度数统计。
  • 建树：用 vector<int> G[N] 存图，并在读入时统计每个节点的出度（子节点个数）。全图的最大出度即为树的度数 KKK。
  • 满树判定：遍历一遍所有出度不为 0 的非叶子节点，如果存在出度 ≠K\neq K=K 的节点，立刻标记 flag = false。
  • 前序遍历：对每个节点的子节点列表 G[i] 进行 sort 排序保证升序。从根节点启动 DFS，将访问顺序存入数组后按格式输出。

L2-052 吉利矩阵

题意

  • 在 N×NN \times NN×N 的矩阵中填入 0∼X0 \sim X0∼X 的数字，要求每行、每列的数字总和都严格等于 XXX。求合法的填法总方案数。

思路

  • DFS+剪枝。
  • 纯粹的 for (0 to X) 暴力填格子会产生 XN2X^{N^2}XN2 的恐怖复杂度，必定 TLE。
  • 剪枝 1（范围压缩）：当前格子最多只能填 min(X - 当前行已填和, X - 当前列已填和)。
  • 剪枝 2（唯一确定）：当填到当前行的最后一列或当前列的最后一行时，该格子的数值已经唯一确定（必须等于 X−X -X− 剩余总和）。不需要进行 for 循环枚举，只需验证填入该数后是否合法，合法则直接进入下一层 DFS。
  • 实测只要做到剪枝2就不会T，只做剪枝1会T一个

L3-037 夺宝大赛

题意

  • 网格图中有一个唯一的宝藏。KKK 个玩家分布在各处，同时以最短路径走向宝藏。
  • 获胜条件：某一批玩家到达时间相同，且在此时间没有其他并列者，且这是所有无并列者的时间中最短的一个。求最后的获胜者。

正解思路

  • 逆向单源 BFS。宝藏点是唯一的，以宝藏点为起点全图跑一次 BFS，求出宝藏到所有格子的最短时间 dist[x][y]。查询每个玩家只需 O(1)O(1)O(1) 读表。
  • 赛场发病写的正向，TLE

L3-038 工业园区建设

题意

  • 极度繁琐的工业园区规划问题。大致为图论中选择若干核心建设点，考虑周边污染与路径连通成本，最小化某个代价函数（本质考察树哈希与子树同构状态压缩）。

正解思路

  • 树哈希 + 状态压缩 DP。利用树哈希识别子树同构，从而极大地压缩 DP 的状态空间

💥 骗分战术 (图论玄学万能药)

  • 看到“修路”、“代价最小”、“图论建设”，如果你读不懂题，或者不会写状态转移：
    • 骗分套路：写一个 O(2N)O(2^N)O(2N) 的暴力组合枚举。用 DFS 选或者不选每个节点，计算合法代价并打擂台求最小。稳拿 N≤10N \le 10N≤10 的测试点，又白嫖 10 分！

L3-039 攀岩

题意

  • 机器人在无尽平面的岩点上攀岩，机械臂有长度限制。规划路径，使得整个过程中机械臂所需的最大长度尽可能小。
  • 本质是求路径上三元组外接圆直径的最大值最小化。

正解思路

  • 计算几何 + 图论。需要 Delaunay 三角剖分，外加最短路/最小生成树瓶颈路。完全是代码量上百行的神仙题。

💥 骗分战术

  • 既然是求路径上“最大边权的最小值”，这不就是图论里的瓶颈路模型吗？
  • 不要管什么三角形外接圆，直接暴力建图！算出任意两个岩点之间的欧几里得直线距离作为边权。
  • 在这张巨大的图上，直接跑一遍 Dijkstra 或者 Kruskal 求最小瓶颈路。

题意

  • 两个人玩卡牌游戏,每个人手中有nnn张牌,所有牌的数字恰好构成一个长为2n2n2n的排列
  • 游戏规则如下
    • 如果一个人手中没有牌了,游戏结束
    • 比较两个人手牌中最前一张,数字大的一方获得一分并移除该张牌,另一方不得分
  • 在开始前,你可以任意的改变你的手牌顺序,求出有多少种排列能使你获得最大分数

思路

  • 观察发现,对于自己手中的牌,可以分为两类:能得分的,不能得分的

  • 能得分的牌想得分就一定要放到不能得分的牌前面
  • 所以整个数组分为两部分能得分的∣不能得分的{\text{能得分的}|\text{不能得分的}}能得分的∣不能得分的,在这两部分内部,数字可以任意排列

题意

  • 给定一个长为nnn的序列aaa,表示运算式:a1+a2+a3..+ana_1+a_2+a_3..+a_na1​+a2​+a3​..+an​,问有多少种把+替换成|的方式,使得不改变运算式的值(不替换也是一种方案,且特别的:本题中认为|运算优先级大于+)

思路

  • 妙妙题
  • **|运算优先级大于+**意味着可以把数组切分成若干连续|块,块与块之间用加号链接
  • 然后求解划分的方案数
  • 对于这类问题通常枚举最后一段的位置,然后计算
  • 同类典型例题
    • n个数字之间,加入k个*使得构成的算式值最大
    • 设dp[i][j]dp[i][j]dp[i][j]表示前iii个数字放了jjj个乘号最大价值
    • 转移方程dp[i][j]=max(dp[k][j−1])∗[k+1,i]dp[i][j]=max(dp[k][j-1])*[k+1,i]dp[i][j]=max(dp[k][j−1])∗[k+1,i]
  • 本题因为没有规定要划分多少段,所以不需要第二维
  • 定义dp[i]dp[i]dp[i]表示[1,i][1,i][1,i]这个子数组中,替换的合法方案数
  • 则从iii往前找连续的一段,保证这一段求和的结果和求或的结果一样,记这一段为[j,i][j,i][j,i]
  • 那么从iii往前最多到jjj都可以成为最后的连续|段,对应的方案数就是dp[j−1]...dp[i−1]dp[j-1]...dp[i-1]dp[j−1]...dp[i−1]
  • 那么就可以转移

    dp[i]=∑k=j−1i−1dp[k]dp[i]=\sum_{k=j-1}^{i-1}dp[k] dp[i]=k=j−1∑i−1​dp[k]

  • 用一个前缀和来优化则

    dp[i]=dps[i−1]−dps[j−2]dp[i]=dps[i-1]-dps[j-2] dp[i]=dps[i−1]−dps[j−2]

    dps[i]=dps[i−1]−dp[i]dps[i]=dps[i-1]-dp[i] dps[i]=dps[i−1]−dp[i]

  • 对于每一个iii怎么快速的获得jjj的位置?
    • 读入完后跑一遍双指针

补充知识:划分型dp

  • 给定一个序列,要求将其切分成若干个连续的子段,使得这些子段满足某些条件,或者让某种总代价（最大值/最小值/计数）达到最优

  • 几乎所有的划分型 DP 都遵循同一个套路:枚举最后一段,划分型 DP往往需要用一些手段来优化枚举(前缀和…)

  • 状态定义:dp[i]dp[i]dp[i]表示前iii个元素(下标 1…i1 \dots i1…i)切分完毕后,所能达到的最优解(或方案数)

  • 转移方程:

    • 为了算出 dp[i]dp[i]dp[i],我们枚举最后一段的起点（假设最后一段是 j+1j+1j+1 到 iii）

    dp[i]=∑0≤j<i(dp[j] ⊕ Cost(j+1,i))dp[i] = \sum_{0 \le j < i} \left( dp[j] \ \oplus \ \text{Cost}(j+1, i) \right) dp[i]=0≤j<i∑​(dp[j] ⊕ Cost(j+1,i))

    • jjj:枚举的切分点（上一段的结束位置）
    • dp[j]dp[j]dp[j]:前 jjj 个已经切好的最优结果
    • Cost(j+1,i)\text{Cost}(j+1, i)Cost(j+1,i):最后一段 [j+1,i][j+1, i][j+1,i] 的属性,比如和、最大值、是否合法等,注意是(j+1)(j+1)(j+1)
    • ⊕\oplus⊕:连接操作（如果是求方案数就是 +,求最大值就是 max,求和就是 +）

题意

  • 给定一个序列,其中有白色数组也有黑色数字,再给定参数k表示一开始可以选择k个白数字染红
  • 接下来每秒都会发生:所有红色数字都会把其右侧aia_iai​,个数字里的白色数字染红
  • 求：以最优策略染红最初的k个白色数字的话,所有白色数字都会被染红的最短时间
  • (注意：黑色数字不会、也无需被染红)

思路

  • 显然,答案具有单调性,超过某一个时间后都是合格的,在这个时间之前都是不合格的
  • 考虑二分,二分最终状态不合格...r|l...合格的,答案就是二分的lll
  • 对时间进行二分,写check检验当前时间下填满需要多少次,用次数和k比较
  • 怎么检验
    • 贪心的角度考虑,第一个白色是一定要染的,否则就永远无法它
    • 染色完第一个后,就看第一个能传播多少,等传播够对应的秒或者传播不动了就染下一个
    • 重复这个过程
  • 如何快速的获得aia_iai​最远传播到多少
    • 对于每个白点,染色它能传播到的最远距离是ai+ia_i+iai​+i
    • 所以,对于第iii个位置,能到达的最远距离其实就是对a_j+j \, (j\leqi)做前缀最大

题意

  • 给定一个区间[l,r][l,r][l,r],可以进行如下操作任意次
  • 选取区间中若干个数字(至少一个),将他们的按位与加入集合SSS
  • 求:SSS的mex最大是多少
  • mex:未出现在集合中的最小非负整数

思路

  • 情况1:注意到如果l==0l==0l==0,那么所有小于rrr的数都可以通过只选取一个数构造出来,而由于与运算只会变小,所以大于rrr的数一定无法构造出来,答案是r+1r+1r+1
  • 情况2:注意到如果l,rl,rl,r最高位相同,那么区间中的任何数做与,最高位都一定是1,0无法被构造出来,答案是0
  • 情况3:继续按照最高位考虑,记b1,b2b_1,b_2b1​,b2​为l,rl,rl,r的位数,如果b2−b1>=2b_2-b_1>=2b2​−b1​>=2,那么100...000和001...111两个数一定在区间内,用前者到后者之间的每一个数和后者做与,就能够造出001...111 0001...111~0001...111 0,所以这种情况下答案也是r+1r+1r+1
  • 情况4:
    • 如果b2−b1==1b_2-b_1==1b2​−b1​==1,那么10...0~r和01...1做与可以得到r−2b2 0r-2^{b_2}~0r−2b2​ 0,也就是答案至少是r−2b2+1r-2^{b_2}+1r−2b2​+1
    • 注意到,lll本身是可以向下延伸的,一定可以延伸到从高往低数,第一个0后全填充0的状态
    • 因为从第一个0后,全是1的数到全是0的数一定存在,把第一个0改成1后面全是0的数也一定存在,两者做与就可以向下拓宽lll
    • 把算出来的下界,和至少的ans做一下合并就ok了

题意

  • 给定mmm条信息,[l,r,k][l,r,k][l,r,k],表示一个排列在[l,r][l,r][l,r]的最大值是kkk
  • 求长度为nnn且符合所有信息的排列的个数

思路

  • 注意到两个性质
    • 对于数字kkk,只能放到所有对应区间的交集
    • 对于数字kkk,所有区间的并集的最大值就是kkk了
  • 对于性质2,kkk越小优先级越高,也就是每个位置放的数的上界是其对应的最小的kkk
  • 相应的,对于数字而言,数值越大的限制越多,因为他不能出击所有比自己小的区间
  • 所以优先填放数值大的数字
  • 综上
    • 我们需要先从小到大处理所有区间,获得每个位置可以填的最小上界lim[i]lim[i]lim[i]
    • 然后,用lim[i]lim[i]lim[i]维护维护每个值xxx的并集位置G[x]G[x]G[x]
    • 最后从大到小枚举每个值可能出现的位置,检查在交集中的个数cntcntcnt,ans∗=cntans*=cntans∗=cnt
    • 注意,还要同步维护一个spacespacespace,表示没有任何限制的位置的个数,每次确定一个数后,它剩余的可能位置就被释放,要加给spacespacespace
  • 复杂度瓶颈在于第一步,如果暴力的做有可能退化到O(qn)O(qn)O(qn)
    • 我们希望每个位置被最小值覆盖后就不再被访问
    • 从小到大枚举区间
    • 维护一个类似nxtnxtnxt的东西,只要被覆盖了就给覆盖区间内所有位置的nxtnxtnxt标记为区间右界

    • 然后维护的时候就可以跳跃访问,每次从i->fa[i]
    • 复杂度O(n)O(n)O(n)

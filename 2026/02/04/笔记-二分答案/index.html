二分答案 (Binary Search for Answer)

1. 思想

将 “求解问题” 转化为 “判定问题”。

  • 求解：直接算出答案是多少（难）。
  • 判定：给一个值 mid，判断它是否合法（易，通常配合贪心）。

前提：答案必须具有单调性（如果 xxx 合法，那么比 xxx 小/大的数也都合法）。

2. 适用场景

题目中出现以下字眼时，99% 是二分答案：

  • “使最大值最小”
  • “使最小值最大”
  • “求满足条件的最小/大值”

3. 模板 (while(l <= r))

图解

  • ans的可取范围通常如下图所示

…,是,是,是⏟左侧区域∣否,否,否,…⏟右侧区域\underbrace{\dots, \text{是}, \text{是}, \text{是}}_{\text{左侧区域}} \quad \mid \quad \underbrace{\text{否}, \text{否}, \text{否}, \dots}_{\text{右侧区域}} 左侧区域…,是,是,是​​∣右侧区域否,否,否,…​​

  • while(l <= r) 循环，本质上是在不断压缩 l 和 r，试图把那条分界线夹出来。

    • l ：不断往右挤，试图跨过分界线进入右边。
    • r ：不断往左挤，试图跨过分界线进入左边。

…,是,r⏟左侧最大值∣l,否,…⏟右侧最小值\underbrace{\dots, \text{是}, \mathbf{r}}_{\text{左侧最大值}} \quad \mid \quad \underbrace{\mathbf{l}, \text{否}, \dots}_{\text{右侧最小值}} 左侧最大值…,是,r​​∣右侧最小值l,否,…​​

  • 当 while(l <= r) 结束时，条件 l > r 成立。
    此时，l 和 r 的位置关系一定是：l 刚好在 r 的右边一位。即 l = r + 1。
    • r：停在了左侧（满足条件）区域的最后一个数。
    • l：停在了右侧（不满足条件）区域的第一个数。

4. check(mid) 函数怎么写？

  • 逻辑：假设答案是 mid，我能不能利用现有的资源/条件达到要求？
  • 返回值：true (合法/满足条件) / false (不合法)。
